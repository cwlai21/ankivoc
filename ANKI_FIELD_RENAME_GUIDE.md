# Anki Field Renaming Guide

## 問題說明

當你更改解釋語言時(例如從英文改為法文),卡片欄位會顯示為空。這是因為 Anki 模型的欄位名稱是固定的,需要更新為通用名稱。

## 解決方案

我們已經更新了程式碼和資料庫,使用通用的 `Explanation` 欄位名稱,這樣同一個模型就可以支援任何解釋語言。

## 需要手動操作的部分

**AnkiConnect 不支援自動重命名欄位**,所以你需要在 Anki 中手動重命名欄位。

### 步驟 1: 重命名 中文-(R/L) 模型欄位

1. 開啟 Anki
2. 點選: **工具 → 管理筆記類型** (Tools → Manage Note Types)
3. 選擇: **中文-(R/L)**
4. 點選 **欄位...** (Fields...) 按鈕
5. 重命名下列欄位:
   - `English` → `Explanation`
   - `exemple-EN` → `exemple-Explanation`
   - `exemple2-EN` → `exemple2-Explanation`
6. 點選 **確定** 儲存

### 步驟 2: 重命名 Français-(R/L) 模型欄位

1. 在**管理筆記類型**視窗中
2. 選擇: **Français-(R/L)**
3. 點選 **欄位...** 按鈕
4. 重命名下列欄位(如果存在的話):
   - `English` → `Explanation` (如果有)
   - `exemple-EN` → `exemple-Explanation`
   - `exemple2-EN` → `exemple2-Explanation`
5. 點選 **確定** 儲存

## 驗證

重命名完成後:

1. 在資料庫中創建一個新的批次,目標語言選中文,解釋語言選法文
2. 添加一些詞彙並處理
3. 在 Anki 中檢查新卡片,應該會顯示法文翻譯和例句
4. 你也可以切換回英文解釋,應該同樣正常工作

## 疑難排解

### Q: 重命名後舊卡片會怎樣?
A: 舊卡片會自動使用新的欄位名稱,內容保持不變。Anki 的重命名是安全的,不會遺失數據。

### Q: 如果我不想手動重命名怎麼辦?
A: 你可以選擇刪除 Anki 中的模型(會刪除所有卡片!),然後讓系統重新創建新模型。但這會遺失學習進度。

### Q: 我有很多卡片,重命名會很慢嗎?
A: 不會,欄位重命名是即時的,與卡片數量無關。
